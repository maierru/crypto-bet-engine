## Codebase Patterns
- Spring Boot 3.4.4 with Java 21, Gradle Kotlin DSL
- Tests use H2 (PostgreSQL mode) + local redis-server process on port 16379
- EmbeddedRedisConfig starts `redis-server` via ProcessBuilder (embedded-redis lib doesn't work on ARM Mac)
- Test profile: `@ActiveProfiles("test")`, config in `application-test.yml`
- Flyway disabled in tests, using `ddl-auto: create-drop` with H2
- All monetary values: `BigDecimal` in Java, `DECIMAL(19,4)` in PostgreSQL
- Package: `com.cryptobet.engine`
- UUID primary keys everywhere
- All `@SpringBootTest` classes MUST include `@AutoConfigureMockMvc` to share context and avoid embedded Redis port conflicts
- Monetary amounts serialized as strings via `@JsonFormat(shape = STRING)` with `setScale(2, RoundingMode.HALF_UP)` in response DTOs
- Always use `setScale(n, RoundingMode.HALF_UP)` — never bare `setScale(n)` which throws ArithmeticException on non-exact values
- Paginated endpoints: use Spring Data `Page<T>` return type with `PageRequest.of(page, size)` — returns `content`, `totalElements`, `totalPages`, `number`
- Wallet-scoped bet listing: `GET /api/wallets/{id}/bets?status=X&page=0&size=20` on WalletController, delegates to BetService
- Wallet balance mutations use pessimistic locking via `findByIdForUpdate()` (SELECT FOR UPDATE)
- Feature packages: `com.cryptobet.engine.<feature>` with `dto` sub-package for request/response records
- Configurable properties use `@Value("${betting.prop-name:default}")` pattern
- OddsService is injectable — use it wherever odds/payout calculation is needed
- Redis exposure keys: `exposure:{symbol}` — use ExposureService for all exposure mutations
- ExposureService uses `StringRedisTemplate` with get/set (not INCRBYFLOAT) for BigDecimal precision
- PriceService stores latest prices in ConcurrentHashMap — use `getPrice(symbol)` for current price
- BinancePriceFeed connects to Binance via Java HttpClient WebSocket — disable in tests with `binance.symbols: ""`
- Binance config: `binance.ws-url` and `binance.symbols` (CSV) in application.yml
- `@Scheduled` methods must NOT call `@Transactional` methods on `this` (self-invocation bypasses proxy) — split scheduler + service into separate beans
- `@EnableScheduling` on main application class
- Settlement: SettlementScheduler (scheduler) calls SettlementService (transactional per-bet logic)
- Bet entity has `resolveAt` (expiry), `priceAtResolution`, `resolvedAt` fields for settlement
- BetStatus: OPEN, WON, LOST, PUSH, CANCELLED
- PlaceBetRequest accepts optional `durationSeconds` (defaults to `betting.default-duration-seconds: 60`)
- WebSocket: STOMP over `/ws` endpoint, topics `/topic/prices` (all) and `/topic/prices/{symbol}` (per-symbol)
- PriceService publishes `PriceUpdateEvent` via Spring ApplicationEventPublisher on every price update
- PriceWebSocketHandler listens for PriceUpdateEvent and broadcasts JSON to STOMP topics
- Settlement scheduler disabled in tests via `settlement.scheduling-enabled: false` to avoid cross-context interference
- Tests using `RANDOM_PORT` share H2 in-memory DB — disable background schedulers to prevent test pollution
- Redis keys must be cleaned in `@BeforeEach` when tests assert on Redis state (not just DB)
- Global error handling via `@ControllerAdvice` in `com.cryptobet.engine.error.GlobalExceptionHandler` — no local `@ExceptionHandler` on controllers
- Error response format: `{"error":"CODE","message":"...","details":{...}}` via `ErrorResponse` record
- Bean Validation on all request DTOs: `@NotNull`, `@Positive`, `@NotBlank` etc. with `@Valid` on controller `@RequestBody`
- Insufficient balance throws `InsufficientBalanceException` (409 Conflict), not `IllegalArgumentException`
- Validation errors return 400 with `VALIDATION_ERROR` code and field-level details map
- Production profile: `application-prod.yml` with env var placeholders, HikariCP pool config, graceful shutdown
- Structured JSON logging: `logback-prod.xml` with `JsonEncoder` (built into logback 1.5.x, no extra deps)
- Docker: multi-stage Dockerfile (temurin:21-jdk-alpine build → temurin:21-jre-alpine runtime), non-root user
- CI: GitHub Actions `.github/workflows/ci.yml` — build+test with Redis service, then Docker image build
- EmbeddedRedisConfig `isPortAvailable()` check allows tests to use CI-provided Redis without conflict
