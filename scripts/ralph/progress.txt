## Codebase Patterns
- Wallet entity uses `@PrePersist`/`@PreUpdate` for timestamps — no need to set manually
- DTOs are Java records; adding fields requires updating all constructor calls in tests
- WalletResponse uses `@JsonInclude(NON_NULL)` for optional fields — null fields are omitted from JSON
- Flyway migrations are in `src/main/resources/db/migration/` with `V<n>__` naming
- Tests use `@SpringBootTest` + `@AutoConfigureMockMvc` + `@ActiveProfiles("test")` with H2 in-memory DB
- WalletService accepts primitives (not DTOs) — controller unwraps request records before calling service
