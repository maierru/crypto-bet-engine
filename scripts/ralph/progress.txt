## Codebase Patterns
- E2E tests use Ferrum (Ruby CDP) + Minitest, located in test/e2e/
- test_helper.rb provides E2EHelpers module — include it and get setup/teardown + all wait/action helpers
- Browser is headless by default, set HEADLESS=false to watch visually
- APP_URL env var overrides base URL (default http://localhost:8080)
- Screenshots auto-saved to test/e2e/screenshots/ on test failure
- application-e2e.yml sets bet duration to 10s for faster settlement tests
- Use `fill_input` helper for text inputs (clears then types), `select_option` for dropdowns (sets value + dispatches change)
- On Linux, Ferrum auto-adds no-sandbox browser option
- To test JS validation, remove HTML5 validation attrs (minlength, min, required) via browser.evaluate() first
- renderPriceCard in app.js clears only .skeleton elements (not all grid children) when first real price arrives
- App needs Docker rebuild to pick up static file changes (no volume mount)
- Ferrum native `.click` may fail on buttons below the fold or behind overlays — use `@browser.evaluate("document.querySelector('selector').click()")` for reliable JS clicks
- "Settling..." text in countdown is very transient (~1s) — wait for either "Settling" OR "No active bets" to avoid timing misses
- Bet outcome is non-deterministic with real market data — don't assert specific WIN/LOSS; verify settlement completed (no active bets) instead
- e2e profile must be active for 10s bet duration; without it, default is 60s — use docker-compose override to set SPRING_PROFILES_ACTIVE=prod,e2e
